## Common Pitfalls to Avoid

### ❌ Pitfall 1: Memorizing without understanding
```
WRONG: "Quicksort is O(n log n)"
RIGHT: "Quicksort averages O(n log n) because partition splits array in half. 
        Worst case O(n²) when pivot is always min/max. 
        We use random pivot to avoid this."
```

### ❌ Pitfall 2: Ignoring space complexity
```
WRONG: "Merge sort is best because O(n log n)"
RIGHT: "Merge sort is O(n log n) time but uses O(n) extra space. 
        Quick sort uses O(log n) stack space. Trade-off matters."
```

### ❌ Pitfall 3: Not applying to real problems
```
WRONG: "I learned binary search today"
RIGHT: "I learned binary search. Applied it to fix MongoDB query that was doing full scan. 
        Search now 10x faster with index."
```

### ❌ Pitfall 4: Using built-ins while learning
```
WRONG: "I implemented sort using Array.sort()"
RIGHT: "I implemented quicksort from scratch. 
        Here's my partition function, my pivot selection, my recursive calls."
```

### ❌ Pitfall 5: Not asking "at scale" questions
```
WRONG: "This solution works"
RIGHT: "This solution works. At 1M items it uses 500MB memory and takes 2 seconds. 
        Can we optimize to 100MB and 200ms?"
```

### Q: Do I need to memorize algorithms?
**A:** 
No. You need to UNDERSTAND the flow and mechanics.

If you only memorize, you can't explain or optimize.

### Q: What if I get stuck?
**A:**
1. Ask your AI mentor with specific example
2. Make it more concrete (give actual numbers)
3. Ask "why?" not just "how?"
4. Request step-by-step execution
5. Connect to MERN problem you know

### Q: How do I know I'm ready for FAANG interviews?
**A:**
When you can:
1. Code any algorithm from scratch
2. Explain complexity in detail (with numbers)
3. Optimize real code by 10x
4. Design systems considering algorithms
5. Teach junior developers confidently

### Q: Should I do LeetCode?
**A:**
Yes, but not instead of real project work.
- Real project: 70% (optimize your actual code)
- LeetCode: 20% (pattern recognition)
- Theory: 10% (understand complexity)

### Q: My project doesn't use these algorithms?
**A:**
Every project uses algorithms. You might not see them explicitly:
- Database queries = searching (binary search in index)
- Rendering lists = sorting + filtering
- Real-time features = data structures + algorithms
- Caching = complexity optimization
